# Guía docente

Esta página organiza la guía para usarla en clase, especialmente en segundo de ASIR o como material de refuerzo para administración de sistemas.

---

## Objetivos didácticos

Al finalizar el recorrido, el alumnado debería ser capaz de:

- Usar terminales Linux y Windows con autonomía.
- Interpretar comandos antes de ejecutarlos.
- Administrar archivos, usuarios, permisos, procesos y servicios.
- Diagnosticar problemas de red.
- Revisar logs y eventos.
- Crear scripts básicos y evolucionarlos hacia herramientas mantenibles.
- Documentar procedimientos técnicos.
- Resolver incidencias siguiendo runbooks.
- Usar Git, Docker y Ansible como introducción profesional.

---

## Metodología recomendada

!!! info "Secuencia de trabajo"

    1. Explicación breve del concepto.
    2. Demostración guiada.
    3. Práctica individual.
    4. Puesta en común de errores.
    5. Ejercicio aplicado.
    6. Documentación del procedimiento.
    7. Mejora del ejercicio mediante scripting.

---

## Tipos de actividades

| Tipo | Finalidad |
|---|---|
| Lectura guiada | Entender conceptos y contexto |
| Laboratorio | Practicar comandos en entorno seguro |
| Reto | Resolver sin pasos cerrados |
| Checklist | Verificar aprendizaje y procedimiento |
| Script | Automatizar una tarea repetitiva |
| Runbook | Documentar resolución de una incidencia |
| Proyecto | Integrar varios bloques |

---

## Evaluación sugerida

| Criterio | Peso orientativo |
|---|---:|
| Uso correcto de comandos | 20% |
| Diagnóstico razonado | 20% |
| Scripts funcionales y documentados | 20% |
| Seguridad y buenas prácticas | 15% |
| Documentación y claridad | 15% |
| Autonomía y verificación | 10% |

---

## Rúbrica para scripts

| Nivel | Descripción |
|---|---|
| Inicial | Ejecuta comandos básicos pero sin parámetros ni control de errores |
| Medio | Usa variables, parámetros y salida clara |
| Avanzado | Añade logs, validación, control de errores y documentación |
| Profesional | Es mantenible, seguro, probado, versionado y reutilizable |

## Prácticas recomendadas

1. Inventario de sistema.
2. Diagnóstico de red.
3. Gestión de usuarios y permisos.
4. Comprobación de servicios críticos.
5. Backup con restauración.
6. Revisión de logs.
7. Informe de mantenimiento.
8. Documentación de una incidencia.
9. Script parametrizable.
10. Proyecto final.

---

## Consejos para el aula

!!! warning "Seguridad"

    Los comandos destructivos deben probarse solo en máquinas virtuales, laboratorios o entornos preparados.

!!! tip "Aprendizaje real"

    Es mejor entender 20 comandos bien que copiar 200 sin saber cuándo usarlos.

!!! note "Documentación"

    Cada práctica debería terminar con una explicación: qué se ha hecho, por qué, cómo se verifica y qué riesgos existen.
