# Ejercicios prácticos

Estos ejercicios están pensados para practicar de forma progresiva. Se recomienda realizarlos en máquinas virtuales o laboratorios, no en sistemas de producción.

---

## 1. Terminal básica

### Ejercicio 1: navegación

Objetivo: practicar rutas, creación y eliminación de archivos.

Tareas:

1. Crear una carpeta llamada `laboratorio`.
2. Entrar en ella.
3. Crear tres archivos de texto.
4. Listar el contenido.
5. Guardar el listado en un archivo.
6. Buscar una palabra dentro del listado.
7. Eliminar los archivos y la carpeta.

---

## 2. Linux administración

### Ejercicio 2: diagnóstico básico

Objetivo: obtener información del sistema.

Tareas:

1. Mostrar versión del sistema operativo.
2. Mostrar memoria disponible.
3. Mostrar uso de disco.
4. Mostrar procesos con más consumo.
5. Mostrar servicios fallidos.
6. Guardar los resultados en un archivo de texto.

---

## 3. Windows administración

### Ejercicio 3: inventario PowerShell

Objetivo: generar un inventario básico.

Tareas:

1. Mostrar nombre del equipo.
2. Mostrar usuario actual.
3. Mostrar sistema operativo.
4. Mostrar CPU y RAM.
5. Exportar el resultado a CSV.

---

## 4. Redes

### Ejercicio 4: diagnóstico de conectividad

Objetivo: localizar si un problema está en IP, gateway, DNS o puerto.

Tareas:

1. Ver IP local.
2. Ver ruta por defecto.
3. Hacer ping a una IP externa.
4. Resolver un dominio.
5. Comprobar un puerto remoto.
6. Documentar conclusiones.

---

## 5. Usuarios y permisos

### Ejercicio 5: usuario de laboratorio

Objetivo: practicar usuarios, grupos y permisos.

Tareas Linux:

1. Crear un grupo.
2. Crear un usuario.
3. Añadir el usuario al grupo.
4. Crear una carpeta compartida.
5. Asignar propietario y grupo.
6. Ajustar permisos.
7. Comprobar acceso.

Tareas Windows:

1. Crear usuario local.
2. Crear o usar un grupo local.
3. Añadir usuario al grupo.
4. Revisar permisos sobre una carpeta.

---

## 6. Procesos, servicios y logs

### Ejercicio 6: servicio controlado

Objetivo: practicar diagnóstico de servicios.

Tareas:

1. Elegir un servicio no crítico.
2. Consultar su estado.
3. Ver si está habilitado al inicio.
4. Revisar logs o eventos.
5. Reiniciarlo.
6. Verificar funcionamiento.

---

## 7. Backups

### Ejercicio 7: copia con fecha

Objetivo: crear una copia sencilla con nombre fechado.

Tareas:

1. Crear carpeta de origen con varios archivos.
2. Crear carpeta destino.
3. Generar copia comprimida o duplicada.
4. Verificar que existe.
5. Restaurar en otra carpeta.
6. Documentar el procedimiento.

---

## 8. Seguridad

### Ejercicio 8: revisión inicial

Objetivo: hacer una revisión defensiva básica.

Tareas:

1. Listar usuarios.
2. Revisar administradores.
3. Ver servicios activos.
4. Ver puertos abiertos.
5. Revisar firewall.
6. Revisar errores recientes.
7. Escribir recomendaciones.

---

## 9. Bases de datos

### Ejercicio 9: backup y restauración

Objetivo: practicar ciclo básico de base de datos.

Tareas:

1. Crear una base de datos de prueba.
2. Crear una tabla.
3. Insertar datos.
4. Exportar la base.
5. Borrar o recrear la base en laboratorio.
6. Restaurar los datos.
7. Verificar que los datos existen.

---

## 10. Git

### Ejercicio 10: versionar documentación

Objetivo: practicar flujo básico de Git.

Tareas:

1. Crear repositorio local.
2. Crear README.
3. Hacer primer commit.
4. Crear rama.
5. Modificar archivo.
6. Ver diferencias.
7. Fusionar rama.

---

## 11. Docker

### Ejercicio 11: contenedor básico

Objetivo: entender imagen, contenedor, puerto y logs.

Tareas:

1. Descargar una imagen.
2. Ejecutar un contenedor.
3. Listar contenedores.
4. Ver logs.
5. Entrar en el contenedor.
6. Detenerlo y eliminarlo.

---

## 12. Monitorización

### Ejercicio 12: informe de estado

Objetivo: crear un informe manual o semiautomatizado.

Tareas:

1. Revisar CPU.
2. Revisar memoria.
3. Revisar disco.
4. Revisar servicios.
5. Revisar logs.
6. Escribir un resumen con estado y recomendaciones.

---

## 13. Proyecto final inicial

Objetivo: unir varios bloques en una práctica completa.

Crear un procedimiento que:

1. Haga inventario del sistema.
2. Compruebe conectividad.
3. Revise servicios críticos.
4. Revise disco y memoria.
5. Revise logs recientes.
6. Genere un informe.
7. Documente cómo programarlo.

Este proyecto puede evolucionar después hacia Script-Lab.
